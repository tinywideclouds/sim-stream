# @tinywideclouds.com/libs/sim/data-access

## Purpose
This library acts as the **Single Source of Truth** for the Angular application's state. It provides the `SimulationStateService`, a purely reactive, Signal-based store.

By isolating state management in this library, our visual components (like ECharts or Log Tables) remain entirely "dumb." They simply inject this service, read the signals, and automatically re-render when the state changes.

## Architecture: The Signal Graph
The service is divided into three tiers of Signals:

1. **Raw State (`WritableSignal`)**: Holds the arrays generated by `util-parsers` (`actorTimeline`, `powerUsage`, `logs`).
2. **UI State (`WritableSignal`)**: Holds the user's current selections from the Sidebar (`selectedHousehold`, `selectedActors`).
3. **Derived State (`ComputedSignal`)**: Combines the Raw and UI state to generate downstream data. For example, `availableActors` will automatically recalculate to only show actors belonging to the currently `selectedHousehold`.

## Usage
```typescript
import { SimulationStateService } from '@[tinywideclouds.com/libs/sim/data-access](https://tinywideclouds.com/libs/sim/data-access)';

export class MyChartComponent {
  private state = inject(SimulationStateService);

  // Automatically updates if new data is loaded OR if the user changes the UI filters!
  filteredTimeline = computed(() => {
    return this.state.actorTimeline().filter(row => 
      row.householdId === this.state.selectedHousehold() &&
      this.state.selectedActors().includes(row.actorId)
    );
  });
}
```
